# Apps Script (SaaSポータル) セットアップ手順

本システムでは、Google Apps Script (GAS) を利用して、AI（Gemini）を搭載した「リッチな権限申請Webポータル」と、承認・棚卸しを行う「スプレッドシート管理画面」を提供します。

## 事前準備（Step 0）

システムのフロントエンドおよびデータベース（表示用）の土台となるスプレッドシートを作成します。

1. **スプレッドシートの新規作成**: Googleドライブ等から、新しいスプレッドシートを作成し、任意の名前（例：「Cloud Access Manager 管理表」）を付けます。
1. **GASエディタを開く**: 作成したスプレッドシートのメニューバーから `拡張機能` > `Apps Script` をクリックし、エディタ画面を開きます。
1. **コードの配置**: リポジトリの `apps-script/` 配下にある以下の3つのファイルを、GASエディタ上にすべて作成し、コードをコピー＆ペーストして保存します。
   - `Code.gs` （バックエンド処理・スプレッドシート制御）
   - `GeminiRoleAdvisor.gs` （WebアプリのエントリポイントとAI連携）
   - `RoleAdvisor.html` （Webアプリのフロントエンド画面）

## Step 1. スクリプトプロパティの設定

GASがGCPのバックエンド（Cloud RunやBigQuery）と通信するための環境変数を設定します。

ターミナルで以下のコマンドを実行し、登録すべき値（JSON）を確認します。

```bash
bash scripts/sync-config.sh
cat apps-script/script-properties.json
```

出力されたJSONの内容を、GASエディタ画面の **左側メニュー「プロジェクトの設定（歯車マーク ⚙️）」 > 一番下の「スクリプト プロパティを追加」** から、1つずつ手動で登録してください。

- `BQ_PROJECT_ID`
- `BQ_DATASET_ID`
- `BQ_LOCATION`
- `CLOUD_RUN_EXECUTE_URL`
- `GAS_INVOKER_SA_EMAIL`

## Step 2. GCPプロジェクトの紐づけとAPI・権限の有効化

GASプロジェクトを、構築したGCPプロジェクトと紐づけ、必要な権限を付与します。

### 2-1. GCPプロジェクト番号の紐づけ

ターミナルで以下のコマンドを実行し、プロジェクト番号（数字の羅列）を取得します。

```bash
gcloud projects describe [PROJECT_ID] --format="value(projectNumber)"
```

GASエディタの **左側メニュー「プロジェクトの設定 ⚙️」 > 「Google Cloud Platform (GCP) プロジェクト」セクション** の「プロジェクトを変更」をクリックし、取得したプロジェクト番号を入力して「プロジェクトを設定」をクリックします。

### 2-2. BigQuery APIの追加

GASエディタの **左側メニュー「エディタ `< >`」 > 「サービス」の横にある「＋」ボタン** をクリックし、`BigQuery API` を選択して「追加」をクリックします。

### 2-3. マニフェストファイル（権限）の修正 [重要]

承認メールの送信や各種API連携を許可するため、権限スコープを明示的に指定します。

1. **左側メニュー「プロジェクトの設定 ⚙️」** で、「`appsscript.json` マニフェスト ファイルをエディタで表示する」にチェックを入れます。
1. **「エディタ `< >`」** に戻り、表示された `appsscript.json` に以下の `oauthScopes` を追加して保存してください。（※元のJSONに上書きマージする形になります）

```json
{
  "timeZone": "Asia/Tokyo",
  "dependencies": { ... },
  "exceptionLogging": "STACKDRIVER",
  "oauthScopes": [
    "https://www.googleapis.com/auth/script.external_request",
    "https://www.googleapis.com/auth/bigquery",
    "https://www.googleapis.com/auth/bigquery.readonly",
    "https://www.googleapis.com/auth/cloud-platform",
    "https://www.googleapis.com/auth/spreadsheets",
    "https://www.googleapis.com/auth/script.send_mail"
  ]
}
```

## Step 3. 申請ポータル（Webアプリ）のデプロイ

AI推論機能と申請フォームが一体化した「SaaSポータル（Webアプリ）」を公開します。

1. ご自身のGCPアカウントに **「Vertex AI ユーザー (`roles/aiplatform.user`)」** ロールが付与されていることを確認します（※インフラ構築時に自動付与されているはずです）。
1. GASエディタ右上の **「デプロイ」 > 「新しいデプロイ」** をクリックします。
1. 「種類の選択」の歯車アイコンから **「ウェブアプリ」** を選択します。
   - 実行するユーザー: `自分`
   - アクセスできるユーザー: `組織内のユーザー` または `全員` （要件に応じて選択）
1. 「デプロイ」をクリックし、発行された **ウェブアプリのURL** をコピーします。
   - ※このURLが、今後ユーザーに案内する「権限申請ポータルのURL」になります。社内ポータル等にリンクを掲載してください。

## Step 4. 管理機能のトリガー登録

本構成では、承認結果の適用は `onEdit` ではなく、カスタムメニューから明示実行します。\
そのため、必須トリガーは「定期ステータス同期」のみです。

GASエディタの **左側メニュー「トリガー（時計マーク ⏰）」 > 右下の「トリガーを追加」** から、以下を登録してください。

1. **ステータス自動同期トリガー（推奨）**
   - 実行する関数: `refreshRequestReviewStatus_`
   - イベントのソース: 時間主導型
   - イベントの種類: 分ベースのタイマー
   - 時間の間隔: 15分おき

## Step 5. スプレッドシートのカスタムメニューと初期化

本システムには、スプレッドシート上で運用者が手動でデータ更新やリカバリを行うための「カスタムメニュー」が組み込まれています。

**カスタムメニューの出し方：**
スプレッドシートの画面を\*\*ブラウザでリロード（再読み込み）\*\*してください。GASの `onOpen` 関数が自動で走り、画面上部のメニューバーに **「棚卸し」** という独自メニューが出現します。
※初回実行時は「承認が必要です」という警告ポップアップが出ます。ご自身のアカウントを選択し、「詳細」>「安全ではないページへ移動」から許可を与えてください。

**メニューから実行できる主な操作：**

- **🔄 レビュー結果を一括送信**: `requests_review` で編集した承認/却下を一括適用します。成功行のみ `processed_history` へ移動し、失敗行はエラー理由と共にシートに残ります。
- **未反映の承認済リクエストを再実行**: APIエラーなどで実行に失敗したリクエストを安全に再実行（リカバリ）します。
- **最新ステータス取得**: BigQueryの最新状態を取得し、シート上の申請ステータス（実行結果など）を手動で最新化します。
- **マトリクス更新**: 「IAM権限設定履歴」シート（Connected Sheetsで取り込んだデータ）を元に、全体を俯瞰できるピボットテーブルを自動生成・更新します。
- **不整合アラート(インシデント)の確認**: 野良権限や期限切れ権限などのインシデント一覧をBigQueryから直接抽出し、専用シートに可視化します。

*(※この後、運用手順書(`docs/operation/operations-runbook.md`)に従って、Connected Sheets機能を用いてBigQueryから各種監査ビュー（`v_sheet_*`）をスプレッドシートに取り込んでください)*

## Step 6. 夜間バッチでスプレッドシート自動更新させる

「マスターデータは常に最新であってほしいから、人間がボタンを押す手間すら省きたい」という場合は、GASの標準機能である「トリガー」を使います。

1. GASエディタの左側メニューから **「トリガー（時計のアイコン）」** をクリックします。
1. 右下の **「トリガーを追加」** をクリックします。
1. 以下の設定にして保存します。
   - **実行する関数:** `refreshAllMasterData`
   - **イベントのソース:** 時間主導型
   - **時間ベースのトリガーのタイプ:** 日付ベースのタイマー
   - **時刻を選択:** 午前 3時～4時 （※GCP側の収集バッチが終わっている運用に影響のない深夜帯）

これを設定しておけば、毎朝出社した時には、必ず昨日の最新状態のマスターデータ（プリンシパル・グループメンバー・リソース情報）がスプレッドシートに同期されます。

## トラブルシューティング

- **Q. スプレッドシートに「棚卸し」メニューが表示されない**
  - A. スプレッドシートのブラウザタブをリロード（再読み込み）してください。また、初回起動時の権限承認ダイアログが出たまま裏に隠れていないか確認してください。
- **Q. 申請ポータル（Webアプリ）を開くとエラーになる / 更新が反映されない**
  - A. コードを修正した場合は、「デプロイ > デプロイの管理」から右上の鉛筆マーク（編集）をクリックし、**必ず「新バージョン」を選択して再デプロイ**する必要があります。
- **Q. Webアプリから申請してもBigQueryに書き込まれない / 承認してもCloud Runが動かない**
  - A. まず、承認処理はシートのステータスを変更した後に**必ずカスタムメニュー「🔄 レビュー結果を一括送信」を実行**しているか確認してください（自動では反映されません）。それでも動かない場合は、Step 1 のスクリプトプロパティの入力ミスを疑い、GASエディタ左側の「実行数」メニューからエラーログを確認してください。
- **Q. 「Vertex AI API failed」というエラーが出る**
  - A. GCPプロジェクトで `aiplatform.googleapis.com` が有効化されているか、およびGASをデプロイしたユーザーに `roles/aiplatform.user` が付与されているか確認してください。
