# Apps Script セットアップ手順

## 事前準備（Step 0）

設定を始める前に、申請用のGoogleフォームと回答先のスプレッドシートを作成し、リンクさせておく必要があります。

1. **Googleフォームの作成**: 指定の必須項目（後述の「Step 4」参照）を持つフォームを作成します。
1. **スプレッドシートへのリンク [重要]**: フォームの「回答」タブを開き、右上の「スプレッドシートにリンク」アイコン（緑色の十字マーク）をクリックして、回答を記録するスプレッドシートを作成（または既存のものにリンク）してください。
   1. 接続が完了すると、選んだスプレッドシートに 「フォームの回答 1」 といった名前の新しいシート（タブ）が自動的に追加されます。
1. **GASエディタを開く**: リンクして作られたスプレッドシートのメニューバーから `拡張機能` > `Apps Script` をクリックし、エディタ画面を開きます。
1. **コードの配置**: リポジトリの `apps-script/` 配下にある `.gs` と `.html` ファイルを、GASエディタ上にすべてコピー＆ペーストして保存します。

## Step 1. スクリプトプロパティの設定

GASがGCPのバックエンド（Cloud RunやBigQuery）と通信するための環境変数を設定します。

ターミナルで以下のコマンドを実行し、登録すべき値（JSON）を確認します。

```bash
bash scripts/sync-config.sh
cat apps-script/script-properties.json
```

出力されたJSONの内容を、GASエディタ画面の \*\*左側メニュー「プロジェクトの設定（歯車マーク ⚙️）」 > 一番下の「スクリプト プロパティを追加」\*- から、1つずつ手動で登録してください。

- `BQ_PROJECT_ID`
- `BQ_DATASET_ID`
- `BQ_LOCATION`
- `CLOUD_RUN_EXECUTE_URL`
- `GAS_INVOKER_SA_EMAIL`

## Step 2. GCPプロジェクトの紐づけとAPI・権限の有効化

GASプロジェクトを、構築したGCPプロジェクトと紐づけ、必要な権限を付与します。

### 2-1. GCPプロジェクト番号の紐づけ

ターミナルで以下のコマンドを実行し、プロジェクト番号（数字の羅列）を取得します。

```bash
gcloud projects describe [PROJECT_ID] --format="value(projectNumber)"
```

GASエディタの \**左側メニュー「プロジェクトの設定 ⚙️」 > 「Google Cloud Platform (GCP) プロジェクト」セクション*- の「プロジェクトを変更」をクリックし、取得したプロジェクト番号を入力して「プロジェクトを設定」をクリックします。

### 2-2. BigQuery APIの追加

GASエディタの \**左側メニュー「エディタ `< >`」 > 「サービス」の横にある「＋」ボタン*- をクリックし、`BigQuery API` を選択して「追加」をクリックします。

### 2-3. マニフェストファイル（権限）の修正 [重要]

承認メールの送信や各種API連携を許可するため、権限スコープを明示的に指定します。

1. \*\*左側メニュー「プロジェクトの設定 ⚙️」\*- で、「`appsscript.json` マニフェスト ファイルをエディタで表示する」にチェックを入れます。
1. \*\*「エディタ `< >`」\*- に戻り、表示された `appsscript.json` に以下の `oauthScopes` を追加して保存してください。

```json
{
  "timeZone": "Asia/Tokyo",
  "dependencies": { ... },
  "exceptionLogging": "STACKDRIVER",
  "oauthScopes": [
    "https://www.googleapis.com/auth/script.external_request",
    "https://www.googleapis.com/auth/bigquery",
    "https://www.googleapis.com/auth/cloud-platform",
    "https://www.googleapis.com/auth/spreadsheets",
    "https://www.googleapis.com/auth/forms",
    "https://www.googleapis.com/auth/script.send_mail"
  ]
}
```

## Step 3. スプレッドシートのカスタムメニュー機能について

本スクリプトには、スプレッドシート上で運用者が手動でデータ更新を行うための「カスタムメニュー」が組み込まれています。

**カスタムメニューの出し方：**
スプレッドシートの画面を\*\*ブラウザでリロード（再読み込み）\*\*してください。GASの `onOpen` 関数が自動で走り、画面上部のメニューバー（「ヘルプ」の右隣など）に \*\*「棚卸し」\*- という独自メニューが出現します。
※初回実行時は「承認が必要です」という警告ポップアップが出ます。ご自身のアカウントを選択し、「詳細」>「安全ではないページへ移動」から許可を与えてください。

**メニューから実行できる主な操作：**

- **申請反映ステータス更新 (`refreshRequestReviewStatus_`)**:
  ユーザーがこのメニューをクリックすると、BigQueryの最新状態を取得し、シート上の申請ステータス（実行結果など）を手動で最新化します。（※この処理は、Step 5で15分ごとの自動更新トリガーに設定することを推奨します）
- **マトリクス更新 (`refreshIamMatrixPivotFromHistory`)**:
  「IAM権限設定履歴」シートのデータを元に、全体を俯瞰できるピボットテーブルを自動生成・更新します。

## Step 4. 必須のフォーム項目ラベルの確認

システムが正しく動作するためには、Googleフォームの質問タイトル（ラベル）が以下の文字列と**完全に一致**している必要があります。

- `申請種別` （選択肢として「新規付与」「変更」「削除」の他、\*\*「緊急付与」\*\*を用意してください）
- `対象プリンシパル`
- `対象リソース`
- `付与・変更ロール`
- `申請理由・利用目的` または `利用目的` （※未入力は受付不可とするため、必須項目にしてください）
- `利用期限` （日付形式）
- `申請者メール`
- `承認者メール（または承認グループ）`

## Step 5. トリガーの登録

フォーム送信やシート編集を自動検知してプログラムを動かすため、GASのトリガーを設定します。

GASエディタの \*\*左側メニュー「トリガー（時計マーク ⏰）」 > 右下の「トリガーを追加」\*- から、以下の3つを登録してください。

1. **フォーム送信時のトリガー**
   - 実行する関数: `onFormSubmit`
   - イベントのソース: スプレッドシートから
   - イベントの種類: フォーム送信時
   - *(※緊急アクセス(Break-glass)機能も、このトリガーの中で自動的に判定・発火します)*
1. **シート編集時のトリガー**
   - 実行する関数: `onEdit`
   - イベントのソース: スプレッドシートから
   - イベントの種類: 編集時
1. **ステータス自動更新トリガー（推奨）**
   - 実行する関数: `refreshRequestReviewStatus_`
   - イベントのソース: 時間主導型
   - イベントの種類: 分ベースのタイマー
   - 時間の間隔: 15分おき

## Step 6. Gemini提案アシスタント（申請前支援）のデプロイ

申請者が「やりたいこと」を入力するだけで、AIが推奨するIAMロールを提案してくれるWebアプリ機能です。

1. ご自身のGCPアカウントに \*\*「Vertex AI ユーザー (`roles/aiplatform.user`)」\*- ロールが付与されていることを確認します（※インフラ構築時に自動付与されているはずです）。
1. GASエディタ右上の \*\*「デプロイ」 > 「新しいデプロイ」\*- をクリックします。
1. 「種類の選択」の歯車アイコンから \*\*「ウェブアプリ」\*- を選択します。
   - 実行するユーザー: `自分`
   - アクセスできるユーザー: `組織内のユーザー` または `全員`
1. 「デプロイ」をクリックし、発行された \**ウェブアプリのURL*- をコピーします。
1. 申請用Googleフォームの「説明」欄などに、このURLを貼り付け、「どのロールを申請すればよいか分からない場合はこちら」と案内してください。

## トラブルシューティング

- **Q. スプレッドシートに「棚卸し」メニューが表示されない**
  - A. スプレッドシートのブラウザタブをリロード（再読み込み）してください。また、初回起動時の権限承認ダイアログが出たまま裏に隠れていないか確認してください。
- **Q. Gemini提案アシスタント（Webアプリ）を開くとエラーになる**
  - A. デプロイ後にコードを修正した場合は、「デプロイ > デプロイの管理」から新バージョンとして再デプロイ（鉛筆マークの編集から新しいバージョンを作成）する必要があります。
- **Q. フォームから申請してもBigQueryに書き込まれない / 承認してもCloud Runが動かない**
  - A. Step 1 のスクリプトプロパティの入力ミス、または Step 5 のトリガー登録が正しく行われているか確認してください。GASエディタ左側の「実行数」メニューから、エラーのログを確認できます。
